# 知识库自动整理规则（MkDocs 项目）

## 编码规则（最高优先级）
- 所有 Markdown 文件必须按 UTF-8 读取和写入
- 禁止改变文件编码
- 如果发现中文乱码或疑似编码异常，立即停止处理并报告
- 在写入目标文件前，先抽样检查正文前若干行是否为正常中文

## 一、项目说明
本仓库为基于 MkDocs Material 的个人知识库。
目标：将 FlowUs 导出的笔记无损迁移到 `docs/` 并用于发布。

---

## 二、目录结构
- `docs/`：最终发布内容
- `undone/`：待处理笔记（输入区）
- `site/`：构建产物（禁止修改）
- `mkdocs.yml`：导航配置

---

## 三、处理流程（核心）
处理 `undone/` 时必须：
1. 遍历每个笔记文件夹
2. 优先读取该文件夹内 `AGENTS.md`
3. 判断目标目录（docs/下）
4. 执行无损迁移（见规则）
5. 修复图片路径
6. 必要时更新 `mkdocs.yml`
7. 运行 `mkdocs build` 验证
8. 输出变更说明

---

## 四、❗内容保护规则（最高优先级）

### 核心原则
👉 无损迁移（lossless migration），不是内容编辑

---

### 严格禁止
- 改写原文句子
- 优化表达或重写
- 删除内容
- 总结或补充解释
- 改变逻辑结构（除非损坏）
- 引入新信息

---

### 允许操作（仅限以下）

#### 1. Markdown 修复（必须）
- 修复标题（#）
- 修复列表/缩进
- 修复代码块（```）
- 修复表格
- 修复公式

#### 2. 路径修复（必须）
- 修复图片路径
- 修复相对路径
- 确保 MkDocs 可渲染

#### 3. 文件整理（允许）
- 移动到 `docs/` 正确目录
- 重命名 `.md` 文件（简洁语义）
- 重命名图片文件（统一规范）

#### 4. 最小结构修复（仅必要时）
- 修复 FlowUs 导致的格式损坏
- 不改变语义

---

### 一句话原则
👉 只修格式和路径，不动内容

---

## 五、分类规则
默认：
- `docs/ai/`：AI / DL / Transformer / SNN
- `docs/cs/`：系统 / 数据库 / OS
- `docs/tool/`：工具 / 环境
- `docs/essay/`：论文 / 思考

无法判断：
👉 提出建议，不要乱放

---

## 六、图片规则
- 保留所有图片
- 使用相对路径
- 必须可显示
- 可重命名并同步路径

---

## 七、命名规则
- 文件名简洁
- 优先英文
- 避免重复文件
- 若已有文档：优先合并

---

## 八、导航规则
新增页面时：
- 必须更新 `mkdocs.yml`
- 放入正确分类
- 不破坏原结构

---

## 九、导入假设（FlowUs）
- 每个 `undone/xxx/` 为一篇笔记
- 一个主 `.md` + 若干图片
- 自动识别主文件
- 图片视为该笔记资源

---

## 十、安全规则（必须）
- 禁止访问仓库外文件
- 禁止读取 `.ssh`、`.env` 等敏感信息
- 禁止提交敏感内容

---

## 十一、验证
完成后必须：
- ```bash
- mkdocs build


---

## 十二、处理后归档规则

处理完成的笔记：

- 不删除
- 从 `undone/` 移动到 `processed/`
- 保持原始结构不变
- 不修改原始导入内容

目的：
- 保留原始数据用于回溯
- 避免重复处理


## 十三、FlowUs资源文件处理规则

对于导入结构：

- 主 Markdown 文件 + 同名资源文件夹

必须执行：

1. 将资源文件夹重命名为简洁名称
2. 将 Markdown 文件同步重命名
3. 更新所有图片引用路径
4. 使用相对路径（./xxx/image.png）

CSS 文件：
- 默认忽略
- 不迁移，除非明确影响内容展示

目标：
👉 生成结构清晰、可维护的 Markdown + 图片目录
